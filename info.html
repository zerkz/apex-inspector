Apex Inspector Glossary/Information

Boxcarred Requests

A boxcarred request is a Salesforce network optimization technique that batches multiple actions into a single HTTP request. This reduces network overhead and improves performance, especially for Lightning/Aura applications.

  • Multiple Apex actions are sent together in one request.
  • Responses are grouped and returned together.
  • Boxcarred requests are visually indicated in the Apex Inspector UI.

For more details, see the Asynchronous Processing guide.

Apex Error Handling

Apex Inspector parses and displays Salesforce Apex errors, including exception details and stack traces. Errors are highlighted in the UI and shown in the expanded response panel for easier debugging.

  • Supports multiple Salesforce error formats (error, errors, exceptionType, stackTrace).

Performance Summary Table

The Salesforce provided Performance Summary table displays timing and resource usage for each Apex request, including total and database time for each action. This helps identify slow or resource-intensive operations, especially if your requests were boxcarred.

  • Shows top-level summary fields (version, request, actionsTotal, overhead).
  • Breaks down each action with total and DB time.
  • Displayed side-by-side with timing data in the expanded row.
